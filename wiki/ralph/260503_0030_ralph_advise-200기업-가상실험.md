# advise_vote 200 기업 가상 실험 (deterministic + consistency)

`advise_vote_before_meeting` tool에 동일 회사 / 동일 질문을 여러 번 던지고 응답이 일관되는지 측정.

## 가정 (가상환경 명시)
- **컨텍스트 모름**: LLM은 이 프로젝트의 사전 지식 / 대화 히스토리 / 메모리 X (No current conversation context)
- **웹검색 X**: Naver/Google/뉴스 등 외부 검색 X (no web search)
- **MCP only**: 오직 OPM 17 tool만 사용
- **as if it's the first question**: 매 호출이 "처음 받은 질문"인 것처럼
- **deterministic (temperature=0)**: 같은 입력에 같은 출력이 나와야 함

→ "MCP가 single source of truth로 충분한가" + "결정 logic이 deterministic한가" stress test.

- 회사 dedupe → 첫 50 회사 (rcept_dt desc)

## Fixed question (모든 회사 동일)

**질문**: "이 회사의 다가오는 주주총회 안건에 대해 어떻게 의결권을 행사해야 하는가?"

**호출 형태** (deterministic — 인자 모두 명시):
```
build_advise_vote_payload(
    company={ticker_or_name},
    year=2025,
    meeting_type="annual",  # 임시는 "extraordinary"
    vote_style="open_proxy",
    enable_marco=False,  # 속도 우선, Marco 별도 spot
)
```

## Fixed output schema (모든 결과 행)

`/tmp/advise_200_results.csv`:
```
ticker,market,company,canonical_name,year,meeting_type,run_no,
status,elapsed_s,
agenda_count,for_count,against_count,review_count,
candidates_count,
revenue_krw,roe_pct,debt_ratio_pct,capital_impairment_status,
audit_opinion_latest,
score_for_pct,score_against_pct,score_review_pct,
warnings_first
```

**Score 정의**:
- `score_for_pct = for_count / agenda_count * 100`
- `score_against_pct = against_count / agenda_count * 100`
- `score_review_pct = review_count / agenda_count * 100`

**Decision 정의** (회사 단위):
- 회사 결정 = 최다 빈도 카테고리 (FOR / AGAINST / REVIEW)
- 동률 시 우선순위: AGAINST > REVIEW > FOR (보수적)

## 반복 횟수

각 회사를 **3회씩** 동일하게 호출 (run_no = 1, 2, 3).
- 200 회사 × 3 run = 600 호출
- 6 worker 병렬 → 예상 ~30분 (회사당 17초 × 600 / 6 worker / 30 min)

DART 호출 cache는 deterministic이지만, shareholder_meeting v2 검색 → director_evaluation fallback 등 분기 logic이 일관되는지 측정.

---

## Consistency 분석 (실험 후)

### Per-company consistency
- run 1, 2, 3의 회사 결정 (FOR/AGAINST/REVIEW)이 동일한 비율
- 100% (모두 동일) / 67% (2건 동일) / 33% (모두 다름)
- target: **≥ 95% 회사 100% 일치**

### Score variance
- 각 회사의 `score_for_pct` / `score_against_pct` / `score_review_pct`의 3 run stddev
- target: **< 5% stddev** (deterministic이라면 ~0)

### Unstable cases
- 결정이 다르거나 score variance ≥ 5% 회사 list
- 원인 분석: DART API 응답 변동 / shareholder_meeting v2 검색 noise / Marco lookup 변동
