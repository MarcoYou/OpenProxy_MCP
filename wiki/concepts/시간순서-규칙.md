# 시간순서 규칙

## 개념

AGM 데이터의 시간적 인과관계를 보호하는 규칙. 미래 데이터로 과거 데이터를 오염시키는 것을 방지.

## 시간 흐름

```
소집결의 (이사회) -> 소집공고 -> 주총 당일 -> 주총결과
```

## 규칙

- 공고 데이터를 결과에서 참조 -> OK (공고가 먼저 존재)
- 결과 데이터를 공고에 넣기 -> **금지** (시간 역전)

## 거버넌스 분석에서의 의미

소집공고 시점에 투표 결과를 알 수 없으므로, 의안분석 시 결과를 참조해서는 안 됨. v4 JSON에서 voteResults는 별도 섹션으로 분리.
