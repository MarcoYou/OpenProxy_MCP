# 기업가치제고계획 (밸류업)

## 개요

2024년 정부의 "기업 밸류업 프로그램"에 따라 상장사가 자율적으로 제출하는 기업가치 제고 계획 공시. 저PBR 기업의 주주환원 정책, ROE 목표, 자본 활용 전략 등을 담는다.

## 소스

- **제출처**: [[DART-OpenAPI]] + [[KRX-KIND]] (병행 제출 가능)
- **공시 유형**: pblntf_ty=I (거래소공시) / KIND 세부 코드 0184
- **DART API**: 전용 구조화 API 없음. `list.json` (pblntf_ty=I) + 제목 키워드 매칭
- **KIND API**: `kind_search_value_up()` — 기업가치제고(0184) 세부검색
- **report_nm 패턴**: `기업가치제고계획(자율공시)`, `기업가치제고계획`, `기업가치제고 관련 안내`
- **의무 여부**: **자율** (법정 의무 아님)
- **공시 주기**: 수시 (연 1회 ~ 분기마다 다양, 기업별)

## 공시 단계 (OPM 분류)

밸류업 공시는 크게 3단계로 구분:

1. **예고공시** (Pre-announcement): 계획 수립 착수 안내. 본 계획 아직 없음.
2. **본계획** (Plan): 실제 목표 수치, 정책 제시. 가장 중요한 단계.
3. **이행 점검** (Progress Review): 본계획 대비 진척도 보고.

OPM의 `value_up` tool은 이 3단계를 자동 분류.

실무상 DART `report_nm`만으로는 단계가 부족하다. 표준 공시 제목은 같은
`기업가치제고계획(자율공시)`여도 본문 `1. 계획서 명칭`에
`2025년 ... 기업가치 제고계획 이행현황`처럼 실제 문서 성격이 드러나는 경우가 있다.
OPM은 `계획서 명칭`을 category 보정 단서로 사용한다.
단, `기업가치제고계획예고(안내공시)`는 실제 계획 본문이 없는 예고 단계이므로
`pre_announcement`로 별도 분류한다.

또한 `2. 주요 내용` 안에 이행현황과 이행전망이 함께 들어가는 경우가 많다.
OPM은 본문 단락을 다음 tag로 나누어 노출한다.

| tag | 의미 |
|---|---|
| `implementation_status` | `이행현황`, `이행 현황`, `이행내역`, `이행 내역`, `진행 현황`, 실적 비교, 진행률 |
| `implementation_outlook` | 이행전망, 예상, 추진/예정 수치 |
| `implementation_result` | 명시적 `이행결과` |
| `future_plan` | 향후 계획, 배분원칙, 중장기 정책 update |
| `meta_reference` | 고배당기업 표시, 기존 공시 참조, 재공시 안내 |

고배당기업 표시 재공시는 category를 `meta_amendment`로 유지하되, 본문 안에
업데이트된 주주환원 결과가 있으면 `embedded_results`와 `latest_result`로 별도 보존한다.
전수 비교 결과 meta 재공시는 최신 공시일 수 있어도 최신 이행현황을 대체하는
기본 데이터로 보기는 어렵다. 기본 분석 패키지는 본계획(`latest_plan`)과 최신
이행현황(`latest_status`)을 중심으로 만들고, `meta_amendment`는 고배당기업
표시/조세특례 reference 및 내장 결과 업데이트만 보조로 결합한다.

Plain language: 밸류업 공시는 "무엇을 하겠다는 계획"과 "지금 어디까지 했는지"를
같이 봐야 한다. `latest_plan`은 계획, `latest_status`는 진행상황이다.
`이행결과`는 항상 따로 존재하지 않으므로, 명시적으로 발견될 때만 `latest_result`로
꺼내 보여주고 없으면 `null`로 둔다.

## 전체 문서 구조 (트리)

표준 서식이 느슨함. 기업별 양식 차이 큼. 공통 항목:

```
[회사명] 기업가치제고계획(자율공시)
│
├── 1. 회사의 현황 진단
│   ├── PBR, ROE, PER 등 주요 지표 추이 (3-5년)
│   ├── 업종 평균 대비 비교
│   └── 저평가 요인 자체 분석
│
├── 2. 목표 수립 (Numerical Goals)
│   ├── ROE 목표 (예: "3년 내 10% 달성")
│   ├── PBR 개선 목표
│   └── 주주환원율 목표 (총환원율)
│
├── 3. 주주환원 정책
│   ├── 배당성향 목표 (예: "당기순이익의 30% 이상")
│   ├── 자사주 매입 계획
│   ├── 자사주 소각 정책
│   └── 배당 정책 (분기배당 도입 여부)
│
├── 4. 자본 효율성 개선 방안
│   ├── 사업 포트폴리오 조정
│   ├── M&A 전략
│   ├── 비핵심 자산 매각
│   └── 중장기 R&D / CAPEX 계획
│
├── 5. 거버넌스 개선
│   ├── 이사회 다양성 / 독립성 강화
│   ├── 여성·외국인 사외이사 확대
│   ├── 전자투표 도입
│   └── 주주소통 정책
│
├── 6. 이행 일정 (Roadmap)
│   └── 연차별 실행 계획
│
└── 7. 이행 점검 체계
    └── 정기 점검 주기 (분기/반기/연간)
```

## API 필드 vs 원문 대응

별도 구조화 API 없음. `list.json` 응답 기본 필드만:

| API 필드 | 의미 |
|----------|------|
| `rcept_no` | 접수번호 |
| `corp_name` | 회사명 |
| `report_nm` | 공시 제목 (키워드 매칭) |
| `rcept_dt` | 접수일 |
| `flr_nm` | 제출인 |

**원문 상세**: `get_document()` 또는 KIND 본문 HTML (`kind_fetch_document(acptno)`) → OPM의 `value_up` tool이 텍스트 파싱으로 지표/목표/일정 추출.

## OPM에서의 활용

- **`value_up(scope="summary")`**: 최신 공시 메타 + 단계 분류 (예고/본계획/이행점검)
- **`value_up(scope="plan")`**: 본계획 핵심 내용 (ROE/PBR/배당성향 목표)
- **`value_up(scope="commitments")`**: 정량 목표 추출 (숫자 + 대상기간)
- **`value_up(scope="timeline")`**: 과거 공시 이력 timeline

## 거버넌스 분석 포인트

1. **본계획 vs 예고공시**: 예고공시만 제출한 회사는 **아직 계획 없음**. `value_up(scope="summary")`의 `stage`로 구분.
2. **정량 목표의 명확성**: "노력하겠다" 수준 vs "3년 내 ROE 10%" 같은 구체 수치 — 이행 가능성 차이 큼.
3. **주주환원율 목표**: 배당성향 + 자사주 매입/소각 합산 목표. "총환원율 50%" 같은 통합 지표 제시가 모범.
4. **이행 점검**: 본계획 제출 후 1년 내 진척도 공시가 나오지 않으면 실행 의지 의심.
5. **KIND만 제출 vs DART 병행**: DART에도 제출하면 투자자 접근성 높음. KIND만 제출은 소극적.
