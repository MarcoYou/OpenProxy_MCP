# 자기주식취득결과보고서

자기주식 취득결정 후 **실제 매입 완료 결과**를 보고하는 공시. "결정만" vs "정말 했는지" 검증의 핵심.

## 개요

| 항목 | 값 |
|------|-----|
| 트리거 | 취득기간 종료 |
| 기한 | 종료일 다음날부터 **5일 이내** (자본시장법 시행령 §176조의2 ③) |
| 의무 여부 | 의무 |
| 공시 주체 | 상장법인 |
| 1차 채널 | 금융위(DART) |
| API | **별도 구조화 API 없음** — `list.json` + 키워드 |
| pblntf_ty | A (정기공시 카테고리이나 사건성) |
| report_nm 패턴 | `자기주식취득결과보고서` |
| 본문 파싱 | 일자별 실제 매입 raw + 합계 |

## 핵심 데이터

### list.json 메타
| 필드 | 의미 |
|---|---|
| `corp_code` | 기업코드 |
| `corp_name` | 회사명 |
| `report_nm` | "자기주식취득결과보고서" |
| `rcept_no` | 접수번호 (DART 뷰어 링크) |
| `rcept_dt` | 접수일자 |

### 본문 항목 (예: 삼성전자 rcept_no=20260424000778)
| 항목 | 의미 |
|---|---|
| 주요사항보고서 제출일 | 원래 취득결정 공시일 |
| 취득기간 | 매입 시작일 ~ 종료일 |
| 일자별 매입수량 / 평균단가 / 위탁증권사 | 일별 raw 표 |
| 취득 합계 | 총 주식수, 총 금액 |
| 취득예정금액 미달 여부 | 미달 시 사유 |

## 결정 ↔ 결과 매칭

| 사이클 | 매칭 키 |
|---|---|
| 취득결정 (`tsstkAqDecsn`) → 취득결과보고서 | "주요사항보고서 제출일" 동일 → 같은 사이클 |

## OPM 통합

- `treasury_share` tool에 phase=`execution` 이벤트로 추가 예정
- 결정 ↔ 결과 cross-link으로 "정말 집행했나?" 검증
- 미달 사유 명시 시 warning flag
