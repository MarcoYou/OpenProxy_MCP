# 신탁계약에 의한 취득상황보고서

신탁계약 체결 후 신탁사가 자기주식을 매입하는 **분기별 진행 상황** 정기 보고. 사이클 종료 (해지) 전까지 분기마다 반복.

## 개요

| 항목 | 값 |
|------|-----|
| 트리거 | 신탁 진행 중 분기 종료 |
| 기한 | 분기 종료 후 **5일 이내** |
| 빈도 | 분기 (3·6·9·12월 말 기준) |
| 의무 여부 | 의무 |
| 공시 주체 | 상장법인 |
| 1차 채널 | 금융위(DART) |
| API | **별도 구조화 API 없음** — `list.json` + 키워드 |
| report_nm 패턴 | `신탁계약에의한취득상황보고서` |

## 핵심 데이터

### list.json 메타
| 필드 | 의미 |
|---|---|
| `report_nm` | "신탁계약에의한취득상황보고서" |
| `rcept_no` / `rcept_dt` | 접수번호 / 일자 |

### 본문 항목 (예: 대한조선 rcept_no=20260429000746)
| 항목 | 의미 |
|---|---|
| 신탁계약 체결일 | 원 사이클 시작 |
| 보고기간 | 분기 (예: 2026 Q1) |
| 일자별 취득 raw | 매월/매주 취득 수량 + 금액 + 단가 + 위탁사 |
| 누적 취득 | 사이클 시작 이후 합계 |
| 잔여 한도 | 신탁계약금액 - 누적 취득 |
| 취득예정금액 미달 | 미달 시 사유 |

## 사이클 위치

```
신탁체결 → [상황보고서 Q1] → [Q2] → [Q3] → ... → 신탁해지 → 해지결과보고서
```

## OPM 통합

- `treasury_share` phase=`execution` (반복 발생 분기 보고)
- 같은 신탁 사이클 (체결일 동일) 내 여러 상황보고서 timeline grouping
- 잔여 한도 표시로 향후 매입 여력 추정
