## 개요

상장법인 주식을 5% 이상 보유하게 된 자가 보유 현황과 목적을 신고하는 의무 공시. [[5%-대량보유]] 제도의 실행 문서.

## 소스

- **제출처**: [[DART-OpenAPI]] (DS004 majorstock API)
- **의무 여부**: 의무 (5% 이상 보유 또는 1% 이상 변동 시)
- **공시 주기**: 수시 (5% 이상 보유 시, 이후 1% 이상 변동 시 5일 이내)

## 전체 문서 구조 (트리)

```
주식등의 대량보유상황보고서
│
├── 요약정보 (표지)
│   ├── 발행회사명, 관계
│   ├── 보고구분 (신규/변동/변경)
│   ├── 보유주식등의 수 및 보유비율  ← API stkrt (보고자+특별관계자 합산)
│   ├── 주요계약체결 주식등의 수 및 비율  ← API ctr_stkrt ★ 담보/신탁 주식
│   ├── 의결권의 수 및 보유비율
│   └── 보고사유
│
├── 제1부 보고의 개요
│   ├── 1. 발행회사에 관한 사항 (종목코드, 의결권있는 발행주식 총수)
│   ├── 2. 대량보유자에 관한 사항
│   │   ├── 가. 보고자 (개인/법인 정보, 최대주주 정보)
│   │   └── 나. 특별관계자 목록 (연번, 성명, 구분, 관계, 국적 등)
│   ├── 3. 보유주식등의 수 및 보유비율
│   │   └── 보고자 + 특별관계자 각각의 보유주식수/비율 명세  ← 개별 지분율 여기에만 있음
│   ├── 4. 보유목적 (경영권 영향 / 단순투자 / 일반투자)
│   └── 5. 변동[변경]사유
│
├── 제2부 대량보유내역
│   ├── 1. 보고자 및 특별관계자별 보유내역
│   │   ├── 가. 주식등의 종류별 (주권/전환사채/신주인수권 등)
│   │   └── 나. 보유형태별 (소유에 준하는 보유 제1~7호)
│   ├── 2. 보유주식등에 관한 계약  ← ctr_stkrt의 실체
│   │   └── 신탁·담보·대차·일임·장외매매·공동보유 계약 내역
│   └── 3. 자기계정 및 고객계정별 (금융기관에 한함)
│
└── 제3부 직전보고일 이후 대량변동내역
    ├── 1. 변동내역 총괄표
    ├── 2. 세부변동내역 (취득/처분 일자, 방법, 단가)
    └── 3. 취득에 필요한 자금등의 조성내역
```

## API 필드 vs 원문 대응 (중요)

| API 필드 | 원문 위치 | 실제 의미 | 주의사항 |
|----------|-----------|-----------|----------|
| `stkrt` | 요약정보 > 보유주식등의 수 및 보유비율 | **보고자+특별관계자 합산** 지분율 | 사업보고서 "계"와 일치 |
| `ctr_stkrt` | 요약정보 > **주요계약체결** 주식등의 비율 | 담보·신탁·대차 등 계약이 체결된 주식 비율 | ≠ 보고자 본인 직접보유 |
| `ctr_stkqy` | 동일 | 계약체결 주식 수 | |
| `repror` | 표지 > 보고자 | 보고자 이름 | |
| `report_resn` | 제1부 5. 변동사유 | 변동사유 텍스트 | |
| `report_tp` | 제1부 4. 보유목적 | 경영참여/단순투자/일반투자 | |

**보고자 본인 직접보유 지분율은 API에 없다.**
→ 제1부 3항 원문에서만 파싱 가능 (get_document() 필요).
→ `stkrt`에서 특별관계자 합산분을 빼면 추정 가능하나, 원문 파싱이 정확.

## OPM에서의 활용

- `own_block`: majorstock API + document.xml 원문으로 보유목적 파싱
- `own_full_analysis`: `stkrt`(합산)를 비고에 표시. 보고자 개별 지분은 원문 파싱 필요.
- [[국민연금]], GIC, BlackRock 등 주요 기관투자자의 보유 현황 추적

## 파싱 원칙

새 공시 유형을 다룰 때는 get_document()로 전체 텍스트를 트리로 정리한 뒤,
어떤 섹션에 어떤 데이터가 있는지 확인하고 나서 파싱 로직 작성할 것.
특정 필드가 원하는 값을 담고 있다고 가정하지 말 것.
