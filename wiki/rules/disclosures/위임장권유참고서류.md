## 개요

[[위임장-권유]]를 하려는 자가 주주에게 제공해야 하는 참고 서류. [[프록시-파이트]]의 핵심 공시 문서.

## 소스

- **제출처**: [[DART-OpenAPI]]
- **의무 여부**: 의무 (위임장 권유 시)
- **공시 주기**: 수시 (위임장 권유 시)

## 주요 내용

- 위임장 권유자 정보
- 권유 대상 안건
- 권유자의 의결권 행사 방향 (찬성/반대)
- 주주에게 위임을 요청하는 사유
- 경영진 측 vs 주주 활동가 측 각각 별도 제출 가능

## 전체 문서 구조 (트리)

```
위임장 권유 참고서류
│
├── 제1부 권유자에 관한 사항
│   ├── 권유자 성명 / 법인명
│   ├── 권유자 구분 (회사측 / 주주측)  ← 프록시파이트 여부 판별
│   ├── 권유 목적 (의결권 위임 획득)
│   └── 권유자와 발행회사 관계 (지분율, 보유주식수)
│
├── 제2부 권유 내용
│   ├── 대상 주주총회 (일시, 장소, 안건 목록)
│   ├── 안건별 의결권 행사 방향
│   │   └── 안건번호, 안건명, FOR / AGAINST / ABSTAIN
│   ├── 권유 방법 (우편, 전자, 대리인)
│   └── 권유 비용 및 부담 주체
│
└── 제3부 피권유자 현황
    ├── 대상 주주 범위 (전체 주주 / 특정 주주)
    └── 주주명부 기준일
```

소스: DART (`pblntf_ty="P"` 또는 유사 코드), 프록시 파이트 시 회사측·주주측 각각 별도 제출
→ 동일 주총에 복수의 위임장 권유 참고서류 존재 가능

## API 필드 vs 원문 대응

| DART 필드 | 원문 위치 | 실제 의미 | 주의사항 |
|-----------|-----------|-----------|----------|
| `rcept_no` | 표지 | 접수번호 | 동일 주총 복수 서류 구분 시 사용 |
| `flr_nm` | 표지 > 제출인 | 권유자 명칭 | 회사측이면 법인명, 주주측이면 주주명 |
| `rcept_dt` | 표지 | 접수일자 | 주총일 2주 전후 발행 패턴 |
| `report_nm` | 표지 | 공시 제목 | "위임장권유참고서류" 포함 여부 확인 |
| 안건별 행사방향 | 제2부 의결권 행사 방향 | FOR/AGAINST/ABSTAIN | **API 직접 제공 안 함** — 원문 파싱 필요 |
| 권유자 구분 | 제1부 > 권유자 구분 | 회사측 vs 주주측 | 원문에서 텍스트 판별 필요 |

**안건별 의결권 행사 방향은 API 필드로 제공되지 않는다.**
→ `get_document()`로 XML 원문 취득 후 제2부 섹션 파싱 필요.
→ 동일 주총에 복수 제출 시 `rcept_no` 순서와 `flr_nm`으로 회사측/주주측 구분.

## OPM에서의 활용
