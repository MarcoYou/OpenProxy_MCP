# TSR — Total Shareholder Return (글로벌 정의)

## 정의

```
TSR = (P_end - P_start + Dividends) / P_start × 100
```

- P_start: 사업연도 첫 거래일 종가 (분모 = 시작 주가)
- P_end: 사업연도 마지막 거래일 종가
- Dividends: 1주당 연간 배당금 합계 (DPS — 결산 + 분기 합산)

**주식 투자자 관점**의 1주 수익률. 주가 변동(자본이득) + 배당(인컴)의 합계를 시작 주가로 나눈다.

## CSR(한국식) vs TSR(글로벌)

같은 "주주환원"이라는 단어를 쓰지만 정의가 다르다.

| 지표 | 분자 | 분모 | 관점 |
|---|---|---|---|
| **CSR** (한국식) | 배당 + 자사주 매입 | 지배주주 당기순이익 | 회사가 돌려준 현금 (회사 회계) |
| **TSR** (글로벌) | 주가 변동 + 배당 | 시작 주가 | 1주 보유 시 수익률 (투자자 관점) |

CSR은 회사가 번 돈 중 얼마나 환원했는지, TSR은 그 회사 주식 1주를 사면 얼마나 벌었는지.

## 데이터 소스

| 컴포넌트 | 1차 소스 | 폴백 |
|---|---|---|
| P_start | 네이버 금융 (siseJson) | KRX Open API |
| P_end | 네이버 금융 (siseJson) | KRX Open API |
| DPS | DART `alotMatter.cash_dps` (사업보고서) | 현금배당결정 공시 합산 |

비거래일이면 네이버 API가 7일 윈도우로 직전/직후 거래일 종가를 자동 폴백 (`_naver_stock_price`).

## 검증 결과 (2026-04-27)

### KT&G 2024

| 항목 | 값 |
|---|---|
| P_start (2024-01-02 시점 종가) | 89,300원 |
| P_end (2024-12-30 시점 종가) | 107,100원 |
| DPS (연간 합계) | 5,400원 |
| 주가 변동률 | +19.93% |
| 배당 수익률 (P_start 기준) | 6.05% |
| **TSR** | **25.98%** |

### 삼성전자 2024

| 항목 | 값 |
|---|---|
| P_start | 79,600원 |
| P_end | 53,200원 |
| DPS (4분기 합산) | 1,446원 |
| 주가 변동률 | -33.17% |
| 배당 수익률 | 1.82% |
| **TSR** | **-31.35%** |

주가 하락폭이 크면 TSR이 음수일 수 있다 — CSR로는 보이지 않는 신호.

### 삼성전자 2025

| 항목 | 값 |
|---|---|
| P_start | 53,400원 |
| P_end | 119,900원 |
| DPS (분기 합산) | 1,668원 |
| 주가 변동률 | +124.53% |
| 배당 수익률 | 3.12% |
| **TSR** | **127.66%** |

V자 반등으로 2024 -31.35% → 2025 +127.66%. 같은 회사가 2년 연속으로 CSR(38.10% → 40.71%)은 안정적인데 TSR은 격차가 큰 사례.

## CSR vs TSR 4-Quadrant Matrix (2024 vs 2025)

| 회사 / 연도 | CSR | TSR |
|---|---|---|
| KT&G 2024 | 92.21% | 25.98% |
| 삼성전자 2024 | 38.10% | -31.35% |
| 삼성전자 2025 | 40.71% | 127.66% |

해석:
- KT&G 2024: 회사가 적극 환원 + 주가도 상승. 매우 양호
- 삼성전자 2024: 회사는 환원했지만 주가 폭락 (반도체 사이클·환율 등 외부 요인)
- 삼성전자 2025: 환원 일관성 + 주가 V자 반등

## 안정성 가드

- 종목코드 누락 시 `ratio_status="missing_price_data"` + warning
- P_start 또는 P_end가 None이면 ratio 계산 안 하고 components만 노출
- 비거래일 자동 폴백 (네이버 7일 윈도우)
- 음수 이익(적자) 회사도 TSR은 산출 가능 (분모가 P_start이므로)
