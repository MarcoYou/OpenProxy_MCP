## 개요

주주총회 개최 전 주주에게 일시, 장소, 안건을 공지하는 의무 공시. OPM AGM 분석의 기반 원본 문서.

## 소스

- **제출처**: [[DART-OpenAPI]] (pblntf_ty=E)
- **의무 여부**: 의무
- **공시 주기**: 정기 (정기주총 전 2주 이상 공고)

## 주요 내용

- 주총 일시/장소
- 안건 목록 (이사 선임, 재무제표 승인, [[정관변경]], [[보수한도]] 등)
- 후보자 정보 (경력, 추천인)
- 재무제표 요약
- [[자사주]] 처분/소각 안건

## 전체 문서 구조 (트리)

```
주주총회 소집공고
│
├── 공고 개요 (표지)
│   ├── 회사명, 주총 일시 및 장소
│   └── 공고일 (주총일 2주 이상 전)
│
├── 제1호 의안 ~ 제N호 의안
│   ├── 재무제표 승인 (대차대조표, 손익계산서, 이익잉여금처분계산서)
│   ├── 이사 선임
│   │   └── 후보자별 성명, 생년월일, 주요 경력, 추천인
│   ├── 감사위원 선임 (해당 시)
│   │   └── 후보자별 성명, 경력, 사외이사 여부
│   ├── 이사 보수한도 승인  ← agm_compensation 소스
│   │   └── 이사 총 보수한도액
│   └── 정관 변경 (해당 시)  ← agm_aoi_change 소스
│       └── 조항별 현행 내용 / 변경 내용
│
└── 참고서류
    ├── 후보자 인적사항 (경력 세부 사항)  ← agm_personnel 소스
    └── 이사회 의결 내역 (후보 추천 근거)
```

API: DART `pblntf_ty="A"` (소집공고), `agendaNo` / `agendaTitle` / `agendaContent` 파싱 필요
3-tier fallback: XML 원문 → PDF (Upstage OCR) 순서로 시도

## API 필드 vs 원문 대응

| API 필드 | 원문 위치 | 실제 의미 | 주의사항 |
|----------|-----------|-----------|----------|
| `pblntf_ty` | - | 공시 유형 코드 | `"A"` = 소집공고 |
| `rcept_no` | 표지 | 접수번호 | 문서 원문 조회 시 사용 |
| `corp_cls` | 표지 | 법인 구분 (Y/K/N/E) | |
| `report_nm` | 표지 | 공시 제목 | "주주총회소집공고" 포함 여부 확인 |
| `flr_nm` | 표지 | 제출인 (회사명) | |
| `rcept_dt` | 표지 | 접수일자 | 주총일과 다름 — 주총일은 원문에서 파싱 필요 |
| `agendaNo` | 제N호 의안 | 안건 번호 | XML 파싱 필요, API 직접 제공 안 함 |
| `agendaTitle` | 제N호 의안 제목 | 안건명 | XML 파싱 필요 |
| `agendaContent` | 의안 본문 | 안건 세부 내용 | XML 파싱 필요 |

**주총 일시, 장소, 후보자 경력은 API 필드로 직접 제공되지 않는다.**
→ `get_document()`로 XML/PDF 원문을 가져와 파싱 필요. [[3-tier-fallback]] 전략 적용.

## OPM에서의 활용

[[agm-tool-rule]]에 정의된 8개 파서가 이 공시를 [[3-tier-fallback]] 전략으로 파싱:
- agm_agenda: 안건 트리 구성
- agm_financials: 재무제표 추출
- agm_personnel: 후보자 경력 파싱
- agm_aoi_change: [[정관변경]] 조항 추출
- agm_compensation: [[보수한도]]/[[소진율]] 산출
- agm_treasury_share: [[자사주]] 안건 파싱
- agm_capital_reserve: [[감액배당]] 전제 조건 파싱
- agm_retirement_pay: 퇴직금 규정 변경 파싱

파싱 결과는 [[v4-스키마]]의 agendas 배열에 구조화.

## 관련

[[주주총회결과]]는 주총 후 발생하며, [[시간순서-규칙]]에 따라 결과에서 공고를 참조할 수 있으나 역방향은 금지.
