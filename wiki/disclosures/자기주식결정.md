# 자기주식 결정

자기주식 관련 5종 결정 공시를 통합 정리. OPM `treasury_share` tool의 입력 소스.

| 종류 | DART API | pblntf_ty | 비고 |
|------|----------|-----------|------|
| 자기주식 취득결정 | `tsstkAqDecsn.json` | B | 직접 매입 결정 |
| 자기주식 처분결정 | `tsstkDpDecsn.json` | B | 보유 자사주 처분 |
| 자기주식 소각결정 | (별도 API 없음, list.json + keyword) | I | 자본 영구 소각 |
| 자기주식 취득 신탁계약 체결 | `tsstkAqTrctrCnsDecsn.json` | B | 신탁사 위임 매입 |
| 자기주식 취득 신탁계약 해지 | `tsstkAqTrctrCcDecsn.json` | B | 신탁 종료 |

---

## 1. 자기주식 취득결정 (tsstkAqDecsn)

**설명**: 회사가 직접 시장에서 자기주식을 매입하기로 결정. 주주환원 또는 경영권 방어 수단.

**report_nm 패턴**: `주요사항보고서(자기주식취득결정)`

**핵심 필드**:

| API 필드 | 의미 |
|----------|------|
| `bddd` | 이사회결의일 |
| `aqpln_stk_ostk` | 취득 예정 보통주 수 |
| `aqpln_stk_cstk` | 취득 예정 종류주식 수 |
| `aqpln_prc_ostk` | 취득 예정 금액 (보통주) |
| `aqpln_prc_cstk` | 취득 예정 금액 (종류주식) |
| `aqexpd_bgd` | 취득 예정 기간 시작 |
| `aqexpd_edd` | 취득 예정 기간 종료 |
| `hdexpd_bgd` | 보유 예정 기간 시작 |
| `aq_pp` | 취득 목적 (주가안정화, 임직원보상, 경영권방어 등) |
| `aq_mth` | 취득 방법 (장내매수, 장외매수, 공개매수 등) |
| `cs_iv_bk` | 위탁 투자중개업자 (증권사) |

---

## 2. 자기주식 처분결정 (tsstkDpDecsn)

**설명**: 보유 중인 자기주식을 처분(매각, 임직원 부여, 합병대가 등). 우호 지분 만들기에 활용 가능.

**report_nm 패턴**: `주요사항보고서(자기주식처분결정)`

**핵심 필드**:

| API 필드 | 의미 |
|----------|------|
| `bddd` | 이사회결의일 |
| `dppln_stk_ostk` | 처분 예정 보통주 수 |
| `dppln_prc_ostk` | 처분 예정 금액 |
| `dpprpd_bgd` | 처분 예정 기간 시작 |
| `dpprpd_edd` | 처분 예정 기간 종료 |
| `dp_pp` | 처분 목적 (임직원 상여, 자금 조달, 합병대가 등) |
| `dp_mth` | 처분 방법 (장내매도, 시간외, 매각, 부여 등) |
| `dpptncmp_cmpnm` | 처분 상대회사 (제3자 매각 시) |

---

## 3. 자기주식 소각결정

**설명**: 보유 자기주식을 영구적으로 소각하여 발행주식수 감소. 가장 강한 주주환원 신호.

**report_nm 패턴**: `주식소각결정`, `[기재정정]주식소각결정` (※ "자기주식소각결정"이 아닌 "주식소각결정")

**소스**: 별도 구조화 API 없음. `list.json` (pblntf_ty=I) + 제목 키워드 매칭.

**키워드 (OPM 사용)**:
```python
("주식소각결정", "자기주식소각결정", "자사주소각결정", "자기주식소각")
```

**핵심 필드** (list.json 응답):
- `corp_code`, `corp_name`, `report_nm`, `rcept_no`, `rcept_dt`
- 상세 수치는 evidence tool로 원문 다운로드 → DART 뷰어 확인

---

## 4. 자기주식 취득 신탁계약 체결 (tsstkAqTrctrCnsDecsn)

**설명**: 회사가 직접 매입하지 않고 신탁사에 위임해 일정 기간 분산 매입. 주가 영향 완화 + 매입 부담 분산.

**report_nm 패턴**: `주요사항보고서(자기주식취득신탁계약체결결정)`

**핵심 필드**:

| API 필드 | 의미 |
|----------|------|
| `bddd` | 이사회결의일 |
| `cs_iv_bk` | 신탁 수탁자 (증권사 등) |
| `ctr_prc` | 계약 금액 |
| `ctr_pd_bgd` | 계약 기간 시작 |
| `ctr_pd_edd` | 계약 기간 종료 |
| `ctr_pp` | 계약 목적 |

---

## 5. 자기주식 취득 신탁계약 해지 (tsstkAqTrctrCcDecsn)

**설명**: 진행 중인 자사주 신탁계약을 만기 도달 또는 중도 해지.

**report_nm 패턴**: `주요사항보고서(자기주식취득신탁계약해지결정)`

**핵심 필드**:

| API 필드 | 의미 |
|----------|------|
| `bddd` | 이사회결의일 |
| `cs_iv_bk` | 신탁 수탁자 |
| `ctcccr_dt` | 해지일 |
| `ctcccr_rs` | 해지 사유 |
| `aq_stk_ostk` | 신탁 기간 중 취득한 보통주 수 |

## 공통 메타 필드 (전 5종)

| API 필드 | 의미 |
|----------|------|
| `rcept_no`, `rcept_dt` | 접수번호, 접수일 |
| `corp_cls`, `corp_code`, `corp_name` | 법인구분, 기업코드, 회사명 |
| `bddd` | 이사회결의일 |
| `od_a_at_t` / `od_a_at_b` | 사외이사 참석/불참 |
| `adt_a_atn` | 감사 참석 여부 |

## OPM에서의 활용

- **`treasury_share`** (단일 tool, 통합):
  - `scope="events"`: 5종 결정 통합 timeline
  - `scope="acquire"`: 취득결정만
  - `scope="dispose"`: 처분결정만
  - `scope="retire"`: 소각결정만
  - `scope="trust"`: 신탁 체결/해지
  - `scope="annual"`: 사업보고서 기반 연간 누적 (기초/취득/처분/소각/기말)
- **`screen_events`**:
  - `treasury_acquire`, `treasury_dispose`, `treasury_retire` event_type으로 시장 전체 검색
- **`ownership_structure(scope="treasury")`**: 자사주 잔고 스냅샷 (사업보고서 기반)

## 거버넌스 분석 포인트

1. **취득 목적 (`aq_pp`)**:
   - 주가 안정화 → 주주환원 신호 (긍정)
   - 임직원 보상 → 보수 정책 일부 (중립)
   - 경영권 방어 → 우호 지분 확보 (논쟁적)
2. **처분 목적 (`dp_pp`)**:
   - 임직원 상여 → 보상 정책
   - 합병대가 → M&A 거래의 일부
   - **제3자 매각 (`dpptncmp_cmpnm` 채워짐)** → 우호 지분 만들기, 주주가치 희석 우려
3. **소각**: 가장 강한 주주환원. 발행주식수 감소 → EPS 자동 상승.
4. **신탁계약 vs 직접취득**: 신탁은 분산 매입으로 시장 영향 완화. 다만 실집행 비율은 사후 확인 필요 (해지 공시의 `aq_stk_ostk`).
5. **취득 vs 소각의 의미 차이**: 취득만 하고 보유는 잠재적 우호 지분. 소각해야 영구 환원.
