DART API 상태 기록
========================

## 2026-03-23 해결
- 이전에 "차단"으로 판단했던 BadZipFile 에러는 **rcept_no 만료** (정정공고로 대체)
- DART에서 정정공고가 나오면 원본 rcept_no가 무효화됨 → status 013 (접수번호 오류)
- API 자체는 정상 작동 중

## 최신 rcept_no (2026-03-23 기준)
- 삼성전자: 20260312000987 (정정)
- NAVER: 20260305000834 (정정)
- LG화학: 20260224004273
- 케이티앤지: 20260225005779

## 교훈
- BadZipFile = 반드시 차단은 아님. 먼저 응답 body를 확인해서 status 코드 체크할 것
- 정정공고가 나오면 이전 rcept_no는 무효 → 항상 최신 검색 후 사용
