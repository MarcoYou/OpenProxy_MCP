# Case Definitions -- AGM 파서 성공 기준

## 목적

이 문서는 세 가지 역할을 합니다:

1. **AI 판단 기준** -- free-open-proxy에서 AI 어시스턴트가 `agm_guide`를 통해 이 문서를 읽고, 파서 결과가 정상인지 판단하여 사용자에게 fallback을 안내합니다.
2. **LLM fallback 가이드** -- 각 안건이 뭔지, 성공적으로 파싱하면 어떻게 생겨야 하는지의 도메인 설명 + 실제 예시. LLM이 이 문서를 참고하여 예시에 가깝게 파싱하도록 유도.
3. **기계적 품질 판정** -- SUCCESS / SOFT_FAIL / HARD_FAIL 자동 분류 기준. paid-open-proxy 배치 파이프라인에서 자동 fallback 트리거 조건.

## 판정 원칙

**안건 tree 기반 판단**: 해당 안건이 소집공고에 존재하는 경우에만 파서 성공/실패를 판정한다.
안건 자체가 없으면 파서 결과가 비어있는 것이 정상이며, 실패로 분류하지 않는다.

예시:
- 보수한도 승인 안건이 없는 기업 --> `agm_compensation_xml`이 "보수한도 승인 안건이 없습니다" 반환 = **정상**
- 이사 선임 안건이 없는 공기업 --> `agm_personnel_xml`이 0명 반환 = **정상**
- 정관변경 안건이 없는 기업 --> `agm_aoi_change_xml`이 0건 반환 = **정상**

## Fallback 흐름

### free-open-proxy (MCP, 유저 대화형)

```
1. agm_*_xml tool 호출 (빠름)
2. AI가 결과를 아래 판정 기준으로 확인
3. SUCCESS --> 사용자에게 답변
4. SOFT_FAIL --> "파싱 결과가 불완전합니다. PDF로 재시도할까요?" 안내
5. 사용자 동의 --> agm_*_pdf tool 호출 (4초+)
6. 여전히 부족 --> "OCR로 한번 더 시도해볼까요?" 안내
7. 사용자 동의 --> agm_*_ocr tool 호출 (UPSTAGE_API_KEY 필요)
```

AI가 유저에게 안내하는 예시:
- "경력 정보가 한 줄로 뭉쳐있어서 읽기 어렵습니다. PDF 원본에서 다시 파싱하면 깔끔하게 분리될 수 있는데, 시도해볼까요?"
- "재무제표 데이터를 찾을 수 없습니다. PDF 버전에서 다시 시도하면 잡힐 수 있어요."
- "PDF로도 안 잡히네요. OCR(이미지 인식)로 한번 더 시도할 수 있는데, 좀 더 시간이 걸립니다."

### paid-open-proxy (배치 파이프라인, 자동)

```
파서별로 자동 체이닝 (run_pipeline.py):
  XML 파서 --> validate --> SUCCESS면 사용
  SOFT/HARD_FAIL --> PDF 파서 --> better 선택
  여전히 부족 --> OCR fallback --> 사용
  여전히 부족 --> LLM fallback (provider API 토큰)
```

## 파싱 성능 (KOSPI 200, 안건 tree 기반)

| 파서 | XML | PDF | OCR | 비고 |
|------|-----|-----|-----|------|
| agenda | 99.5% | 98.0% | 100% | |

| compensation | 98.4% | 99.5% | 100% | |

## 1. agm_agenda_xml -- 안건 목록

### 이 안건은 무엇인가

주주총회 소집공고의 **회의목적사항** 섹션에서 추출하는 안건 트리입니다. 모든 소집공고에 반드시 존재합니다.

- 제1호, 제2호, ... 형식의 안건 번호
- 세부의안이 있으면 제2-1호, 제2-2호, 제3-1-1호 형식으로 하위 트리 구성
- 각 안건에는 "재무제표 승인", "이사 선임", "정관변경" 등의 제목이 붙음
- 일반적으로 정기주총은 3-8개 안건, 대기업은 5-12개 안건

### 성공 파싱 예시

삼성전자 2026년 정기주총:

```json
[
  {
    "number": "제1호",
    "title": "정관 일부 변경의 건",
    "children": [
      {"number": "제1-1호", "title": "집중투표제 배제 조항 삭제", "children": []},
      {"number": "제1-2호", "title": "개정 상법 반영", "children": []},
      {"number": "제1-3호", "title": "이사의 임기 조문 정비", "children": []},
      {"number": "제1-4호", "title": "주식 소각 조문 정비", "children": []}
    ]
  },
  {
    "number": "제2호",
    "title": "제57기(2025.1.1. ~ 2025.12.31.) 재무상태표, 손익계산서 및 이익잉여금처분계산서 등 재무제표 승인의 건",
    "children": []
  },
  {
    "number": "제3호",
    "title": "사내이사 김용관 선임의 건",
    "children": []
  }
]
```

**핵심 포인트:**
- `number`는 항상 "제N호" 또는 "제N-M호" 형식 (드물게 "제N-M-J호")
- `title`은 안건의 핵심 내용만 포함 (테이블, 조문 텍스트 등이 섞이면 안 됨)
- 세부의안은 `children` 배열에 재귀적으로 포함

### 판정 기준

| 등급 | 기준 | AI 안내 |
|------|------|---------|
| **SUCCESS** | items >= 1, 모든 title 2-150자, number 정규식 정상, 중복 없음 | 정상 답변 |
| **SOFT_FAIL** | title 150-200자, title에 `\|` 포함, 안건 1개만 | "안건 목록이 불완전할 수 있습니다. PDF로 재시도할까요?" |
| **HARD_FAIL** | items 0개, title > 200자, 중복 number | "안건을 파싱할 수 없습니다. PDF로 시도해볼까요?" |

## 2. agm_financials_xml -- 재무제표

### 이 안건은 무엇인가

재무제표 승인 안건의 상세 내용입니다. 정기주총의 제1호 안건인 경우가 많습니다.

- **재무상태표(Balance Sheet)**: 자산/부채/자본의 기말 잔액. 당기/전기 비교.
- **손익계산서(Income Statement)**: 매출/비용/이익 흐름. 당기/전기 비교.
- 연결재무제표(consolidated)와 별도재무제표(separate)가 각각 있을 수 있음
- 단위: 천원, 백만원, 억원 중 하나
- 핵심 계정: 자산총계, 부채총계, 자본총계 (BS) / 매출액(또는 영업수익), 영업이익, 당기순이익 (IS)

### 성공 파싱 예시

삼성전자 2026년 정기주총 (연결 재무상태표):

```json
{
  "consolidated": {
    "balance_sheet": {
      "unit": "백만원",
      "columns": ["account", "current", "prior"],
      "rows": [
        ["자          산", "", ""],
        ["Ⅰ. 유  동  자  산", "247,684,612", "227,062,266"],
        ["1. 현금및현금성자산", "57,856,378", "53,705,579"],
        ["2. 단기금융상품", "67,965,021", "58,909,334"],
        ["3. 단기당기손익-공정가치금융자산", "25,715", "36,877"]
      ]
    }
  }
}
```

**핵심 포인트:**
- `unit`이 반드시 있어야 함 (숫자 해석에 필수)
- `current`/`prior` 컬럼으로 당기/전기 비교
- 숫자는 쉼표 포함 문자열 (예: "247,684,612")
- 실제 원화 값 = 숫자 x 단위 (예: 247,684,612 x 백만원 = 약 247.7조원)

### 판정 기준

| 등급 | 기준 | AI 안내 |
|------|------|---------|
| **SUCCESS** | BS rows >= 5 + IS rows >= 3, unit 존재, 핵심 계정 포함 | 정상 답변 |
| **SOFT_FAIL** | BS만 있고 IS 없음, rows 부족, 핵심 계정 누락 | "재무제표가 부분적으로만 파싱됐습니다. PDF로 보강할까요?" |
| **HARD_FAIL** | BS/IS 모두 None | "재무제표를 찾을 수 없습니다. PDF로 시도해볼까요?" |

## 3. agm_personnel_xml -- 이사/감사 선임

### 이 안건은 무엇인가

이사/감사 선임·해임 안건입니다. 후보자별로 다음 정보를 DART 표준 서식의 테이블에서 추출합니다:

- **성명 + 생년월일**: 후보자 식별
- **직위/카테고리**: 사내이사, 사외이사, 감사위원, 감사 등
- **추천인**: 이사회, 사외이사후보추천위원회 등
- **세부경력(careerDetails)**: 가장 중요한 필드. 각 경력은 **기간 + 내용(회사명+직책)** 으로 분리되어야 함
- **결격사유**: 체납/부실기업/법령상 결격 여부
- **직무수행계획**: 선임 후 어떻게 기여할지
- **추천사유**: 왜 이 후보자를 추천하는지

### 성공 파싱 예시

KT&G 2026년 정기주총, 사외이사 노환용:

```json
{
  "number": "제4호",
  "title": "사외이사 노환용 선임의 건",
  "action": "선임",
  "category": "사외이사",
  "candidates": [{
    "name": "노환용",
    "birthDate": "1956.4.25",
    "roleType": "사외이사",
    "recommender": "KT&G 사외이사후보추천위원회",
    "mainJob": "LG상록재단 비상임이사",
    "careerDetails": [
      {"period": "2021~현재", "content": "現) LG상록재단 비상임이사"},
      {"period": "2022~2024", "content": "前) 경상남도 투자유치자문위원"},
      {"period": "2016~2021", "content": "前) LG연암학원 비상임이사"},
      {"period": "2015~2020", "content": "前) LG전자 상근고문"},
      {"period": "2014~2015", "content": "前) LG전자 B2B부문장, 사장"}
    ],
    "eligibility": {
      "taxDelinquency": "해당사항 없음",
      "insolventMgmt": null,
      "legalDisqualification": null
    }
  }]
}
```

**핵심 포인트:**
- `name`은 한글 2-5자 (영문 병기 시 10자 이내). 절대 조문번호("제31조...")나 안건번호("제2-1호")가 아님
- `careerDetails` 각 항목의 `content`는 **한 줄에 하나의 직책**. 50자 이내가 정상
- 여러 경력이 줄바꿈 없이 합쳐져 100자+ 되면 SOFT_FAIL
- 경력이 0건이면 파싱 실패 (DART 원본에는 반드시 경력이 있음)

### 실패 예시

**career 병합 (SOFT_FAIL):**
```json
{"content": "한화솔라원 CSO한화큐셀 CSO한화솔라원 CCO한화큐셀 CCO㈜한화 전략부문장..."}
```
--> 516자짜리 한 줄. 원래 13개 경력이 분리되어야 함. PDF 파서로 재시도하면 분리 가능.

**이름이 조문번호 (HARD_FAIL):**
```json
{"name": "제 31 조 (이사의 선임) 이사는 주주총회에서 선임한다..."}
```
--> 정관 텍스트가 이름으로 잡힌 케이스.

### 판정 기준

| 등급 | 기준 | AI 안내 |
|------|------|---------|
| **SUCCESS** | appointments >= 1, name 2-10자 정상, careerDetails >= 1건, content <= 100자 | 정상 답변 |
| **SOFT_FAIL** | careerDetails 빈 배열, content > 100자 (병합), birthDate 누락 | "경력 정보가 한 줄로 뭉쳐있습니다. PDF로 분리해볼까요?" |
| **HARD_FAIL** | appointments 0개, name이 조문/안건번호 | "후보자 정보를 파싱할 수 없습니다. PDF로 시도해볼까요?" |

## 4. agm_aoi_change_xml -- 정관변경

### 이 안건은 무엇인가

정관변경 안건입니다. 변경하려는 조항별로 **현행(변경전)**, **변경후**, **변경 사유**를 비교 테이블로 표시합니다.

- 세부의안(제3-1호, 제3-2호 등)으로 나뉘기도 함
- 2025-2026년에는 상법 개정(집중투표제, 이사 충실의무 등)으로 거의 모든 기업이 정관변경 안건 포함
- 변경전 텍스트에는 기존 조문 전문, 변경후에는 수정된 조문 전문 또는 "<삭제>" 표시

### 성공 파싱 예시

LG화학 2026년 정기주총:

```json
{
  "amendments": [{
    "subAgendaId": "",
    "label": "제20조",
    "clause": "제20조",
    "before": "제20조 (이사의 선임) ④ 제1항의 이사 선임 시에는 일주당 일개의 의결권을 가지며, 상법상 집중투표방법은 적용하지 아니한다.",
    "after": "제20조 (이사의 선임) ④ 제1항의 이사 선임 시에는 일주당 일개의 의결권을 가진다.",
    "reason": "집중투표제 배제 금지 반영 (상법 제 542조의7)",
    "additionalClauses": [{
      "clause": "제4조",
      "before": "(신설)",
      "after": "부칙 제4조 (집중투표에 관한 적용례) 제20조 제4항 개정규정은 2026년 9월 10일 이후 최초로 이사의 선임을 위한 주주총회 소집이 있는 경우부터 적용한다.",
      "reason": "개정 상법 시행일 반영"
    }]
  }]
}
```

**핵심 포인트:**
- `before`와 `after` 모두 실제 조문 텍스트여야 함
- `clause`에 `제N조` 형식의 조항 참조가 있어야 함
- `<삭제>`는 정상 (조항 삭제 의미)
- `------` 생략 표기도 정상 (변경 없는 부분 생략, DART 표준 서식)

### 판정 기준

| 등급 | 기준 | AI 안내 |
|------|------|---------|
| **SUCCESS** | amendments >= 1, before/after 5자+, clause에 `제N조` | 정상 답변 |
| **SOFT_FAIL** | clause 누락, reason 누락, subAgendaId 누락 | "정관변경 데이터가 일부 누락됐습니다. PDF로 보강할까요?" |
| **HARD_FAIL** | amendments 0개, before/after 모두 빈값 | "정관변경 사항을 찾을 수 없습니다. PDF로 시도해볼까요?" |

## 5. agm_compensation_xml -- 보수한도

### 이 안건은 무엇인가

이사/감사 보수한도 승인 안건입니다. 거의 모든 정기주총에 존재합니다 (KOSPI 200 중 96%).

DART 표준 서식으로:
- **당기**: 이사의 수(사외이사수) + 보수 최고한도액
- **전기**: 이사의 수(사외이사수) + 실제 지급된 보수총액 + 최고한도액

이사와 감사가 별도 안건인 기업도 있고 (GKL 등), 합산인 기업도 있습니다.

### 성공 파싱 예시

삼성전자 2026년 정기주총:

```json
{
  "items": [{
    "number": "제5호",
    "title": "이사 보수한도 승인의 건",
    "target": "이사",
    "current": {
      "headcount": "8(5)",
      "totalDirectors": 8,
      "outsideDirectors": 5,
      "limit": "450억원",
      "limitAmount": 45000000000
    },
    "prior": {
      "headcount": "10(6)",
      "totalDirectors": 10,
      "outsideDirectors": 6,
      "actualPaid": "287억원",
      "actualPaidAmount": 28700000000,
      "limit": "360억원",
      "limitAmount": 36000000000
    },
    "notes": ["※ 당기(제58기) 보수 한도 총액 450억 : 일반보수 260억, 장기성과보수 190억"]
  }],
  "summary": {
    "totalItems": 1,
    "currentTotalLimit": 45000000000,
    "priorTotalPaid": 28700000000,
    "priorTotalLimit": 36000000000,
    "priorUtilization": 79.7
  }
}
```

**핵심 포인트:**
- `current.limitAmount`가 양수 정수 (원 단위)여야 함 -- 이 숫자가 핵심
- `headcount`에 이사의 수와 사외이사수가 "N(M)" 형식으로 포함
- `prior`에 실제 지급액과 한도가 있어야 전기 대비 소진율 계산 가능
- `target`은 "이사" 또는 "감사"
- 소진율 = 전기 실제 지급 / 전기 한도 x 100

### 판정 기준

| 등급 | 기준 | AI 안내 |
|------|------|---------|
| **SUCCESS** | items >= 1, current.limitAmount > 0, headcount 존재, prior에 금액 데이터 | 정상 답변 |
| **SOFT_FAIL** | current 있지만 prior 빈 dict, limitAmount 없음 | "보수한도 금액을 추출하지 못했습니다. PDF로 시도할까요?" |
| **HARD_FAIL** | items 0개, current 빈 dict | "보수한도 정보를 찾을 수 없습니다. PDF로 시도해볼까요?" |
